Getting Started with Vercel Web Analytics

A comprehensive guide to enable Vercel Web Analytics on your project, install the necessary packages, and start tracking visitor data in minutes.

Framework Support
Next.js, SvelteKit, Remix, Create React App, Nuxt, Vue, Astro, and more
Setup Time
5-10 minutes
Prerequisites

Before you get started with Vercel Web Analytics, make sure you have the following in place:

  • A Vercel account â€” If you don't have one, you can sign up for free
  • A Vercel project â€” If you don't have one, you can create a new project
  • The Vercel CLI installed â€” Install it using your preferred package manager
ðŸ’¡ Note: The Vercel CLI is available for npm, pnpm, yarn, and bun. Choose the command that matches your project's package manager.

Installing the Vercel CLI

Choose your package manager to install the Vercel CLI:

Using pnpm:

pnpm i vercel

Using yarn:

yarn add vercel

Using npm:

npm i vercel

Using bun:

bun add vercel
Setup

Enable Web Analytics in Vercel

To get started, you need to enable Web Analytics for your project on the Vercel dashboard:

  1. Go to the Vercel dashboard
  2. Select your project
  3. Click the Analytics tab
  4. Click Enable from the dialog
ðŸ’¡ Note: Enabling Web Analytics will add new routes (scoped at /_vercel/insights/*) after your next deployment.

Add @vercel/analytics to Your Project

Next, add the @vercel/analytics package to your project using your package manager:

Using pnpm:

pnpm i @vercel/analytics

Using yarn:

yarn add @vercel/analytics

Using npm:

npm i @vercel/analytics

Using bun:

bun add @vercel/analytics
Integration

Add Analytics to Your App

The integration method depends on your framework. Follow the instructions for your specific setup:

Next.js (Pages Directory)

Add the Analytics component to your main app file:

import type { AppProps } from "next/app";
import { Analytics } from "@vercel/analytics/next";

function MyApp({ Component, pageProps }: AppProps) {
  return (
    <>
      <Component {...pageProps} />
      <Analytics />
    </>
  );
}

export default MyApp;

Next.js (App Router)

Add the Analytics component to your root layout:

import { Analytics } from "@vercel/analytics/next";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <head>
        <title>Next.js</title>
      </head>
      <body>
        {children}
        <Analytics />
      </body>
    </html>
  );
}
ðŸ’¡ Note: The Analytics component offers seamless integration with Next.js, including automatic route support for tracking page views.

Remix

Add the Analytics component to your root file:

import {
  Links,
  LiveReload,
  Meta,
  Outlet,
  Scripts,
  ScrollRestoration,
} from "@remix-run/react";
import { Analytics } from "@vercel/analytics/remix";

export default function App() {
  return (
    <html lang="en">
      <head>
        <meta charSet="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1" />
        <Meta />
        <Links />
      </head>
      <body>
        <Analytics />
        <Outlet />
        <ScrollRestoration />
        <Scripts />
        <LiveReload />
      </body>
    </html>
  );
}
ðŸ’¡ Note: The Analytics component includes automatic route detection for Remix, so page views are tracked automatically.

Create React App

Add the Analytics component to your main app file:

import { Analytics } from "@vercel/analytics/react";

export default function App() {
  return (
    <div>
      {/* ... */}
      <Analytics />
    </div>
  );
}
ðŸ’¡ Note: When using the plain React implementation, automatic route support is not available. You may need to use custom event tracking for page changes.

Nuxt

Add the Analytics component to your main component:

<script setup lang="ts">
import { Analytics } from '@vercel/analytics/nuxt';
</script>

<template>
  <Analytics />
  <NuxtPage />
</template>
ðŸ’¡ Note: The Nuxt integration includes automatic route support, so page views are tracked as users navigate your site.

SvelteKit

Call the injectAnalytics function in your main layout:

import { dev } from "$app/environment";
import { injectAnalytics } from "@vercel/analytics/sveltekit";

injectAnalytics({ mode: dev ? "development" : "production" });
ðŸ’¡ Note: The injectAnalytics function automatically handles route tracking for SvelteKit applications.

Vue

Add the Analytics component to your main app component:

<script setup lang="ts">
import { Analytics } from '@vercel/analytics/vue';
</script>

<template>
  <Analytics />
  <!-- your content -->
</template>
ðŸ’¡ Note: Route support is automatically enabled if you're using vue-router in your project.

Astro

Add the Analytics component to your base layout (requires version 1.4.0 or later):

---
import Analytics from '@vercel/analytics/astro';
{/* ... */}
---

<html lang="en">
  <head>
    <meta charset="utf-8" />
    <!-- ... -->
    <Analytics />
  </head>
  <body>
    <slot />
  </body>
</html>
ðŸ’¡ Note: For Astro versions earlier than 1.4.0, configure the webAnalytics property in your astro.config.mjs file instead.

Plain HTML Sites

For HTML sites without a framework, add the following script to your HTML files:

<script>
  window.va = window.va || function () { (window.vaq = window.vaq || []).push(arguments); };
</script>
<script defer src="/_vercel/insights/script.js"></script>
ðŸ’¡ Note: When using the HTML implementation, you do not need to install the @vercel/analytics package. However, automatic route support is not available with this method.

Other Frameworks

For frameworks not listed above, import and call the inject function in your main app file:

import { inject } from "@vercel/analytics";

inject();
ðŸ’¡ Important: The inject function should only be called once in your application and must run on the client side. There is no automatic route support with this method.
Deployment

Deploy Your App

Once you've added the analytics component to your app, deploy it to Vercel:

vercel deploy

If you haven't already, we recommend connecting your project's Git repository, which will enable automatic deployments of your latest commits to the main branch.

Once your app is deployed, it will start tracking visitors and page views automatically.

ðŸ’¡ Note: If everything is set up correctly, you should be able to see a Fetch/XHR request in your browser's Network tab from /_vercel/insights/view when you visit any page.
Monitoring

View Your Data in the Dashboard

After your app is deployed and users have visited your site, you can view your analytics data:

  1. Go to your Vercel dashboard
  2. Select your project
  3. Click the Analytics tab

After a few days of traffic, you'll be able to explore your data by viewing and filtering the analytics panels.

Pro & Enterprise Plans: Users on Pro and Enterprise plans can also add custom events to track specific user interactions such as button clicks, form submissions, or purchases.
